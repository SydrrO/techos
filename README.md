# SYDRRO TECH OS

本项目是一个本地运行的运营面板，用于资产调度、库存管理和订单状态查看。

## 本地运行

1. 首次迁移时可将旧业务数据文件命名为 `data.xlsx`，放在本目录。
2. 双击 `start-local-server.bat`。
3. 浏览器会自动打开 `SYDRRO-TECH-V4.html`（当前版本 V6.1）。

## GitHub Pages

开启 Pages 后可以访问：

`https://sydrro.github.io/techos/`

## 手动录单

本地服务模式下可在页面中使用“手动录单”，补录客户、平台、机器、租期、租金和发货地；编号会按现有档期自动分配，并保存到统一数据源。

## 性能优化

V4.1 会在本地浏览器内缓存订单日期解析、档期冲突表和库存列表；财务图表只在进入“对账统计”时渲染，减少资产调度页的重复计算和卡顿。

V4.2 重点优化视觉层次：页面背景、卡片、表格、按钮、弹窗和状态卡增加轻量动效与更统一的玻璃质感，同时保留原有业务结构。

V4.3 优化发货卡片信息层级：重点突出发货日，到货日、起租日和昵称降级为次要信息，并将昵称独立到下一行显示。

V5.0 升级库存管理：库存设备列表新增“当前占用”和“下一档期”，可直接看到设备是否空闲、出租中客户、预计归还日，以及下一单发货日/起租日/客户平台。

V6.0 取消不稳定的 OCR 图片录单，改为更可靠的手动录单：支持平台选择、日期选择器、自动分配编号，并优化资产占用档期表的对齐、颜色和紧凑日期展示。

V6.1 将订单、库存、发货/归还状态、筛选和自定义机型收敛到本地 SQLite 统一数据源，`data.xlsx` 降级为旧数据迁移和兼容镜像。

## 发货提醒

发货提醒不再固定提前 2 天，而是按南京发货预估运输时间：自提/同城按 0 天，江浙沪皖按 1 天，其它多数地区默认 2 天，新疆/西藏/青海/甘肃/宁夏/内蒙古/黑龙江/吉林/海南等偏远地区按 3 天。所有订单都会先保证“起租前 1 天必须到手”，再按运输天数倒推发货日。

## 数据说明

V6.1 起，本地服务会使用 `sydrro-data.sqlite3` 作为统一数据源。订单、发货状态、归还状态、库存、筛选状态和自定义机型都会保存到同一个 SQLite 数据库。

`data.xlsx` 只作为旧数据迁移入口和兼容镜像；`sydrro-backup.json` 是统一数据源的 JSON 镜像备份。公开仓库会展示已提交的数据文件，上传前请确认里面没有不希望公开的客户或订单信息。
